### Task 1: Fix Post boolean scopes

Rails 6.1 handles boolean casting more strictly. Raw SQL comparisons to `0/'f'` and `1/'t'` should use ActiveRecord boolean queries instead.

**Files:**
- Modify: `app/models/post.rb`

- [ ] **Step 1: Replace the published scope**

In `app/models/post.rb`, replace lines 64-67:

```ruby
  scope :published, -> {
    where("draft = 0 OR draft = 'f'")
      .order(created_at: :desc)
  }
```

with:

```ruby
  scope :published, -> {
    where(draft: false)
      .order(created_at: :desc)
  }
```

- [ ] **Step 2: Replace the drafted scope**

In `app/models/post.rb`, replace lines 73-76:

```ruby
  scope :drafted, -> {
    where("draft = 1 OR draft = 't'")
      .order(created_at: :desc)
  }
```

with:

```ruby
  scope :drafted, -> {
    where(draft: true)
      .order(created_at: :desc)
  }
```

### Task 2: Fix Publication nil-returning scopes

Rails 6.1 is stricter about scope chaining. Scopes that return `nil` when a condition isn't met will break chains like `.sift(term).order(...)`. Every scope must return a relation — use `all` as the fallback.

**Files:**
- Modify: `app/models/publication.rb`

- [ ] **Step 1: Fix the dynamic filter scopes (locations, focusgroups, platforms, fields)**

In `app/models/publication.rb`, replace lines 156-164:

```ruby
  [:location, :focusgroup, :platform, :field].each do |name|
    scope "#{name}s", -> (params) {
      if params.present?
        joins("INNER JOIN #{name}s_publications ON publications.id = #{name}s_publications.publication_id")
        .joins("INNER JOIN #{name}s ON #{name}s.id = #{name}s_publications.#{name}_id")
        .where("#{name}s.description IN (?)", params)
      end
    }
  end
```

with:

```ruby
  [:location, :focusgroup, :platform, :field].each do |name|
    scope "#{name}s", -> (params) {
      if params.present?
        joins("INNER JOIN #{name}s_publications ON publications.id = #{name}s_publications.publication_id")
        .joins("INNER JOIN #{name}s ON #{name}s.id = #{name}s_publications.#{name}_id")
        .where("#{name}s.description IN (?)", params)
      else
        all
      end
    }
  end
```

- [ ] **Step 2: Fix the expired scope**
